## Subject 1: Internet of Things (IoT)

**Q1. Describe Web Communication Protocols for connected devices.**
Ans: These protocols enable IoT devices to communicate over IP networks. Key examples include HTTP and CoAP, which provide request/response interaction models suitable for device state retrieval and remote interaction.

**Q2. What are Message Communication Protocols?**
Ans: Lightweight messaging protocols are optimized for constrained IoT networks. MQTT is widely used for publish/subscribe telemetry, ensuring extremely low bandwidth overhead and reliable delivery in unstable network conditions.

**Q3. Discuss SOAP and REST in IoT.**
Ans: SOAP is an XML-based robust messaging protocol often considered too heavy for standard IoT. REST architectures use simple HTTP verbs over stateless client-server interactions, offering lighter weight overhead ideal for IoT web services.

**Q4. Explain HTTP Restful APIs.**
Ans: They expose dynamic IoT device functionalities directly as web resources, easily controllable via standard GET, POST, PUT, and DELETE HTTP commands, ensuring seamless integration with standard modern web applications.

**Q5. What are Web Sockets used for?**
Ans: Web Sockets provide full-duplex communication channels over a single persistent TCP connection, enabling real-time pushing of crucial IoT sensor data to clients without the huge overhead of repeated HTTP polling requests.

## Subject 2: Artificial Intelligence and Machine Learning

**Q1. What is Machine Learning?**
Ans: Machine Learning is a specialized subset of AI that applies advanced statistical techniques to deliberately give systems the ability to "learn" from broad data and improve their performance on tasks without being explicitly programmed.

**Q2. Outline the core Stages of ML.**
Ans: The general operational stages definitively include targeted data collection, data preparation and structural preprocessing, mathematical model selection, iterative model training, rigorous model evaluation, and eventual active inference/deployment.

**Q3. What are the main Types of Machine Learning?**
Ans: Broadly categorized into Supervised Learning (using labeled target data), Unsupervised Learning (discovering hidden patterns in unlabeled data), and Reinforcement Learning (learning optimal actions via systemic reward/punishment).

**Q4. Explain Cross-validation: K-fold technique.**
Ans: A robust validation method where the dataset is cleanly divided into 'k' uniform subsets. The model is trained iteratively on k-1 subsets and tested on the remaining subset 'k' times to ensure generalizable and robust performance metrics.

**Q5. What are Evaluation metrics like Confusion Metrics?**
Ans: A structural table used to effectively describe the performance of a classification model. It lists True Positives, True Negatives, False Positives, and False Negatives, forming the mathematical basis for calculating precision, recall, and accuracy.

## Subject 3: Compiler Design

**Q1. What are Syntax directed definitions?**
Ans: A context-free grammar whose symbols logically possess attributes and whose production rules consist of semantic computational actions or functions used to carefully compute the values of these attributes uniformly during syntax parsing.

**Q2. Describe the Bottom-up evaluation of S-attributed definitions.**
Ans: An S-attributed definition uses only synthesized attributes. They are easily integrated into standard bottom-up parsers, evaluating structural attributes optimally as rule reductions occur smoothly at the parse tree nodes.

**Q3. Explain L attributes definition.**
Ans: An L-attributed definition structurally allows synthesized attributes and inherited attributes where functional dependency strictly travels from left to right among grammar symbols, seamlessly accommodating predictive top-down parsing techniques.

**Q4. What is Type checking?**
Ans: Type checking rigorously verifies that arbitrary data types are logically used sensibly according to language specific rules. The type system represents variables and operators effectively to strategically catch conceptual programming errors.

**Q5. What is the Overloading of functions and operations?**
Ans: This cleanly allows a single identifier or operator symbol to have multiple diverse implementations seamlessly depending on the specific types of the functional arguments passed, requiring the compiler's semantic analyzer to determine context reliably.

## Subject 4: Natural Language Processing

**Q1. What is Minimum Edit Distance?**
Ans: A computational algorithm deliberately used to quantify exactly how dissimilar two linguistic strings are by aggressively counting the minimum number of functional operations (insertions, deletions, substitutions) smoothly required to transform one.

**Q2. Explain Weighted Automata.**
Ans: A standard finite automaton algorithmically enhanced by logically associating probabilistic weights with sequential transitions, frequently functionally used in robust speech recognition to heavily prioritize the most likely word sequences detected.

**Q3. What are N-grams?**
Ans: A contiguous probabilistic sequence of 'n' functional items from a given sample of structural text or speech. Models logically use n-grams to calculate sequentially the probability of the next word accurately given the previous n-1 words.

**Q4. Discuss HMM and speech recognition.**
Ans: Hidden Markov Models accurately represent mathematical systems that sequentially transition between hidden states over time. In speech recognition, hidden states are structural phonemes and the observable signals are extracted acoustic data frames.

**Q5. Explain Part-of-Speech Tagging.**
Ans: The fundamental linguistic process of deliberately reading contextual text and uniformly assigning exact parts of speech to each structural word (noun, verb), which is undeniably essential for disambiguating word sense based on grammatical context accurately.

## Subject 5: Scientific Aptitude

**Q1. Two unbiased coins tossed. Probability of getting Two Tails?**
Ans: Sample space = {HH, HT, TH, TT} = 4 outcomes. Finding 'TT' occurs 1 time. Probability = 1/4.

**Q2. In a biased coin prob of head is 0.4. Toss 10 times, prob of exactly 3 heads?**
Ans: Use binomial probability formula: P(X=3) = C(10,3) * (0.4)^3 * (0.6)^7 = 120 * 0.064 * 0.02799 = 0.2149.

**Q3. Three coins tossed. Probability of all heads?**
Ans: The sample space has exactly 2^3 = 8 outcomes. The unique event 'HHH' is exactly 1 outcome. Probability = 1/8.

**Q4. Two dice thrown. Probability of sum as 8?**
Ans: Outcomes that accurately sum to 8: (2,6), (3,5), (4,4), (5,3), (6,2). Total 5 distinct outcomes. Total Sample space = 36. Prob = 5/36.

**Q5. One card drawn from 52 cards. Probability it is an ace?**
Ans: A standard deck rigorously has exactly 4 specific aces out of 52 total cards. The expected Probability conceptually is = 4/52 = 1/13.
